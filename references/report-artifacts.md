# 报告产物

`http-test-plus` 每次执行后都会生成一份完整 Markdown 报告。

## 默认位置

- 默认目录：当前工作目录
- 可选覆盖：使用 `--report-dir`
- 文件命名：`<中文模式>-<时间戳>-<中文结果>.md`

## 终端输出

终端只打印：

- 执行模式
- 最终结论
- 总请求数
- 通过/失败统计
- 报告绝对路径

## 报告内容

每份报告包含：

- 总体结论
- 执行统计
- 原始请求地址
- 明文请求头
- 查询参数
- 请求体
- 响应状态码
- 响应头
- 完整响应体
- 提取结果
- 备注和错误信息

## Skill 行为

- 即使用户没有主动要求，也要返回中文结论和报告路径。
- 不主动把完整报告内容粘贴进对话，除非用户明确要求查看。
- 报告标题默认使用中文 `description`，不展示技术 `id`。
